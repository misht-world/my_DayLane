# Changelog

Все заметные изменения DayLane. Формат — по [Keep a Changelog](https://keepachangelog.com/ru/),
версии — по [SemVer](https://semver.org/lang/ru/).

## [1.7.0] — 2026-07-06

### Добавлено
- **Путешествия** — дневник поездки: новый тип дела «Путешествие» (период
  с дневником). Внутри — этапы-подкарточки на день или группу дней: город,
  место (гостиница, музей) и заметки по итогу. Экран «Путешествия» (чемоданчик
  в шапке): сейчас / предстоящие / прошедшие, переход к дате в календаре.
  В календаре полоса поездки помечена чемоданчиком, тап открывает дневник.
- **Места на картах** — у этапа можно сохранить место: «Открыть карты»
  запускает внешнее приложение карт (поиск по названию), а ссылку выбранного
  места можно вставить из буфера («Поделиться → Копировать ссылку» в картах).
  Само приложение остаётся офлайновым.
- **Отмена действий** — плашка «Отменить» после переноса «→ сегодня»,
  «перенести всё», назначения даты отложенному и удаления дела (удаление
  восстанавливается целиком: подпункты, этапы, отметки повторений).
- **Возврат в «Отложенные»** — долгий тап по делу: «В отложенные» (снять
  с дня) и «Удалить», оба с отменой.

### Технически
- Схема БД v5: признак `isTrip` + таблица `trip_stages`; этапы входят
  в экспорт/импорт JSON. Зависимость url_launcher (внешние intent'ы).

## [1.6.0] — 2026-06-23

### Исправлено
- **Уведомления теперь срабатывают.** Раньше время напоминаний считалось в UTC
  (зона устройства не определялась) — напоминания уходили не на то время или
  пропускались. Теперь берётся реальная зона устройства (flutter_timezone),
  напоминания планируются как точные будильники.

### Добавлено
- Календарь следует за выбранной датой: листание дней (в шапке) и листание
  календаря — единое состояние; при переходе через границу месяца календарь
  сам перелистывается.
- Кружок-контур на выбранном дне в календаре (сегодня по-прежнему залит).
- Тап по крупной дате в шапке — выбор любого дня/месяца/года (календарь следует);
  «↺ сегодня» возвращает к текущей дате.
- При добавлении подпункта курсор сразу ставится в новое поле.

### Технически
- Разрешения USE_EXACT_ALARM / SCHEDULE_EXACT_ALARM / RECEIVE_BOOT_COMPLETED;
  запрос точных будильников; зависимость flutter_timezone ^3.0.1.

## [1.5.0] — 2026-06-19

### Добавлено
- Календарь: листание месяцев свайпом влево/вправо; при переходе на соседний
  месяц одна неделя остаётся внахлёст (для непрерывности).
- Раскрытие подпунктов прямо в разделе «Отложенные».
- Чёрный цвет в палитре дел.

### Изменено
- Стиль приложения теперь называется «Ежедневник».
- Календарь упрощён: оставлен только режим «Месяц» (виды «Неделя»/«Две недели»
  убраны).
- Единый стиль секций «Вчера/Сегодня/Завтра/Отложенные»: заглавные названия,
  серая чёрточка у свёрнутых, акцентная у раскрытых; убрана иконка у «Отложенные».
- Карточка дела структурирована по блокам (Когда / Цвет / Подпункты / Примечание)
  с ведущими иконками; поле названия сделано крупным и заметным.
- Разделители между делами — чёрная линия с точкой-кружком слева, линия и под
  последним делом; в списках подпунктов — линии с отступом слева (без точек).
- Иконки приведены к единому скруглённому набору (плюс, стрелки, шестерёнка,
  чекбоксы, галочки).

## [1.4.0] — 2026-06-19

### Добавлено
- Навигация по календарю: стрелки ‹ › и заголовок «Месяц Год» — листание на любой
  период; тап по заголовку открывает выбор месяца/года (сетка 12 месяцев + год),
  чтобы заглянуть на пару месяцев вперёд. Кнопка «↺ сегодня» возвращает к текущей дате.
- Подписи месяцев в сетке: 1-е число помечается коротким названием месяца, дни вне
  выбранного месяца приглушены — чтобы не запутаться на стыке месяцев.
- Экспорт и импорт всех дел в файл JSON (резервная копия) в разделе «Данные» настроек.

### Исправлено
- Логотип выровнен по центру круга (адаптивная иконка).
- Значок у поля «Повторение» приведён к общему стилю карточки.

## [1.3.0] — 2026-06-18

### Добавлено
- Выбор цвета дела в карточке (по умолчанию «Авто» по типу; можно задать свой,
  чтобы выделить дело в календаре).
- Раздел «Отложенные» — дела без даты («ждут своего часа»), свёрнут по умолчанию,
  с быстрым назначением: «сегодня», «завтра», «на дату». Тумблер «Отложить» в карточке.
- Чёрные соединители между связанными делами-периодами в календаре (видна цепочка
  зависимостей), с учётом переноса на следующую неделю.

### Технически
- Схема БД v4: `colorId` (−1 = авто) + поле `deferred`; миграция сбрасывает
  старые цвета в «Авто».

## [1.2.0] — 2026-06-18

### Добавлено
- Повторяющиеся события (третий цвет — фиолетовый): дни рождения, платежи, занятия.
  Типы: каждые N дней/недель/месяцев/лет, по числу месяца (с обрезкой коротких
  месяцев), последний день месяца, за K дней до конца месяца.
- Выполнение повторяющихся — по каждому вхождению (галочка закрывает конкретную дату).
- Напоминания для повторяющихся: планируются ближайшие 12 вхождений.

### Технически
- Схема БД v3: поля повторения + таблица отметок вхождений `recurrence_dones`.
- Доменный модуль `recurrence` под юнит-тестами.

## [1.1.0] — 2026-06-18

### Добавлено
- Логотип/иконка приложения (адаптивная: янтарный фон + три дорожки).
- Долгий тап по дню в календаре — добавить дело на эту дату.
- Выбор «за сколько дней напомнить» (в день / за 1–3 дня / за неделю).

### Изменено
- По умолчанию: светлая тема и вид календаря «Месяц».
- Многодневные полосы показывают название на каждой неделе (а не только на первой).
- Кнопка «Перенести всё на сегодня» — красная, со стрелкой «вниз-назад».

## [1.0.0] — 2026-06-17

Первый релиз v1 (Android, локально, без сети).

### Добавлено
- Главный экран в стиле «Журнал»: дата-герой с навигацией по дням (свайп/стрелки,
  возврат к сегодня), три выделенные сворачиваемые секции **Вчера/Сегодня/Завтра**
  с лентами-заголовками и цветными маркерами.
- Кнопки «+» на секциях Сегодня/Завтра — добавление дела сразу на нужный день.
- Карточка дела: один день / период, всплывающие даты и время, подпункты с
  двусторонней синхронизацией (все подпункты ⇄ дело), примечание.
- Жёсткие каскадные зависимости «Начать после дела» с запретом циклов.
- Перенос невыполненного на сегодня (ручной + опциональный автоперенос).
- Календарь: виды неделя/две недели/месяц, периоды — полосы по дорожкам
  (жадная укладка, лимит дорожек + «+N»), однодневные — точки под числом дня (+N).
- Две задачные краски: однодневные — синий, многодневные — янтарь.
- Локальные напоминания (по умолчанию выключены), светлая и тёмная темы,
  русская локализация.

### Технически
- Flutter + Drift (SQLite) + Riverpod. Доменная логика (scheduling, lanes,
  dependencies, carry_over) покрыта юнит-тестами.
