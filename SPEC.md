# DayLane — техническая спецификация (v1)

Локальное Android-приложение для дел и заметок по делам с акцентом на три горизонта времени (вчера / сегодня / завтра) и наглядный календарь, где многодневные дела показаны полосами.

Платформа v1: **Android** (целевое устройство — Ulefone Armor 27T Pro, крупный экран ~20:9). Без сервера, без аккаунтов, без ИИ. Всё хранится локально.

---

## 1. Идея и принципы

1. **Три разных «когда».** У дела время разложено на независимые понятия, их нельзя смешивать в одно поле «дата»:
   - *плановый день / период* — когда дело живёт в колонках и полосой в календаре;
   - *напоминание* — момент уведомления (опционально, по умолчанию выключено);
   - *время дня* — конкретный час (только у однодневных дел).
2. **Только «Дела».** Никаких отдельных «заметок» как сущности. Примечание — поле внутри дела. Это сознательное упрощение.
3. **Наглядность и простота.** Минимум полей в обычном сценарии, всё сложное (период, привязка) — опционально и спрятано до включения.
4. **Ручной контроль над переносом.** Просроченное не теряется и не уезжает молча (если пользователь сам не включил автоперенос).

## 2. Технологии

- **Flutter** (Dart, stable).
- **Хранилище:** SQLite через **Drift** (реляционная модель: дела ↔ подпункты, дела ↔ зависимости).
- **State management:** **Riverpod**.
- **Уведомления:** `flutter_local_notifications` + `timezone` (локальные напоминания).
- **Локализация:** русский (основной). Заложить `intl`/ARB, чтобы потом добавить английский.
- **Тема:** светлая + тёмная (Material 3, `ThemeMode.system`).

Зависимостей по минимуму. Никаких сетевых вызовов в v1.

## 4. Бизнес-логика

### 4.1. Присутствие дела в дне

Дело «присутствует» в дне `D`, если:
- `kind == single` и `startDate == D`, либо
- `kind == period` и `startDate <= D <= endDate`.

Многодневное дело присутствует **в каждом** своём дне (в колонке показывается «день N из M», где `N = D - startDate + 1`, `M = durationDays`).

### 4.2. Секции «Вчера / Сегодня / Завтра»

Якорь — `today` (локальная дата).
- **Сегодня:** все дела, присутствующие сегодня. Раскрыта по умолчанию.
- **Вчера:** дела, присутствовавшие вчера и **не выполненные** (то, что осталось хвостом). Свёрнута по умолчанию. В заголовке — счётчик невыполненного красным бейджем; если 0 — серое «всё разобрано».
- **Завтра:** дела, присутствующие завтра. Свёрнута по умолчанию. В заголовке — серый счётчик.

Выполненные дела:
- в «Сегодня» остаются зачёркнутыми до конца дня (видно прогресс);
- во «Вчера» не показываются (секция — только про хвост невыполненного).

Подсветка: невыполненные дела с датой/концом периода в прошлом — стиль «просрочено».

### 4.3. Перенос на сегодня

- **Ручной (по умолчанию).** Кнопка «→ сегодня» у каждого вчерашнего дела и «перенести всё на сегодня» сверху секции. Перенос: для `single` ставит `startDate = today`; помечает `carriedOver = true`.
- **Автоперенос (тумблер в настройках, по умолчанию ВЫКЛ).** При запуске приложения, если включено: все невыполненные `single`-дела с прошлой датой переносятся на сегодня и помечаются `carriedOver = true` (значок «↻ перенесено» до конца дня).
- Многодневные дела не «переносятся» — они естественно продолжаются по своему периоду.

### 4.4. Календарь и укладка дорожек (lanes)

- Виды: **Неделя**, **Две недели** (по умолчанию), **Месяц** = скользящие ~30 дней от начала текущей недели вперёд (НЕ календарный месяц). Переключатель — выпадающее меню в заголовке календаря.
- Каждое дело-период рисуется горизонтальной полосой по сетке дней. Дело, пересекающее границу недели, рисуется сегментом на каждой строке-неделе.
- **Укладка в дорожки — автоматическая, жадным алгоритмом:** сортируем периоды по `startDate`; каждый кладём в первую дорожку, где он не пересекается по дням с уже лежащими; иначе создаём новую дорожку. Порядок (кто раньше начинается — выше) — без ручного перетаскивания.
- Сегодняшний день выделен. Тап по дню — фильтр/переход (детали — см. открытые вопросы). Тап по полосе — открыть карточку дела.

### 4.5. Зависимости (привязка «Начать после дела»)

- В карточке дела-периода тумблер **«Начать после дела»**. При включении выбирается родительское дело из списка.
- **Список для привязки:** только дела с датой — многодневные и однодневные **будущие**, кроме самого себя и кроме тех, что создали бы **цикл** (A→B→A). Циклы запрещены, при попытке — не давать выбрать.
- **Поведение привязки (жёсткое, автоматическое):**
  - `startDate(child) = endDate(parent) + 1 день`.
  - Длина сохраняется: `endDate(child) = startDate(child) + durationDays - 1`. Поэтому при привязке пользователь редактирует **Длительность**, а Начало и Конец вычисляются и залочены.
  - Когда `endDate(parent)` меняется (вручную или по своей цепочке) — `child` **сдвигается автоматически и молча**, сохраняя длину. Сдвиг каскадный по всей цепочке.
- **Снятие привязки — только вручную:** крестик у выбранного дела или выключение тумблера. После снятия Начало/Конец снова обычные редактируемые даты.
- Реализовать пересчёт цепочки при любом изменении дат/удалении родителя (если родитель удалён — у `child` привязка снимается, текущие даты сохраняются как фиксированные).

### 4.6. Выполнение

- Галочка у дела переключает `isDone`. У многодневного — закрывает **всё дело целиком**.
- Полосу в календаре при выполнении **не укорачиваем** — пусть видно, когда дело началось и сколько длилось. Выполненное дело в календаре можно отрисовать приглушённым/штриховкой.
- Подпункты: каждый со своей галочкой. На главном экране у дела с подпунктами показывать мини-прогресс «k/n».

### 4.7. Напоминания

- По умолчанию выключены. Опция отдельного дела.
- При включении: правило `atStart` (по умолчанию) / `eachDay` / `atEnd` + время (`reminderTime`, по умолчанию 09:00).
- Для однодневного дела — напоминание на его дату; правило `eachDay`/`atEnd` для `single` эквивалентны `atStart`.
- Планировать/отменять локальные уведомления при сохранении/изменении/удалении дела и при сдвиге по зависимости.

## 5. Экраны

### 5.1. Главный экран
- Шапка: «Дела», текущая дата; справа иконка настроек и кнопка «+» (самая правая, добавление дела).
- Три сворачиваемые секции (Вчера / Сегодня / Завтра) со счётчиками в заголовках. По умолчанию раскрыта только «Сегодня».
- Строка дела: галочка, заголовок, мета-чипы (время, напоминание-колокольчик, бейдж периода «день N из M», мини-прогресс подпунктов). У дела с подпунктами — раскрытие в чек-лист.
- У вчерашних дел — «→ сегодня»; сверху секции «Вчера» — «перенести всё на сегодня».
- Снизу — календарь (по умолчанию «Две недели») с полосами и переключателем вида.

### 5.2. Карточка дела (создание/редактирование)
- Шапка: закрыть (×), заголовок «Дело», «Сохранить».
- Поля: заголовок; сегмент **Один день / Период**.
  - *Один день:* Дата (всплывающий календарь), Время (опционально).
  - *Период:* Начало и Конец (всплывающий календарь) — либо, при включённой привязке, Начало (залочено, «после дела»), **Длительность** (−/+), Конец (вычислен, залочен). Тумблер «Начать после дела» + блок выбора родительского дела с подсказкой о каскадном сдвиге.
- Напоминание (тумблер, по умолчанию выкл; при вкл — правило + время).
- Цвет в календаре (палитра-кружки).
- Подпункты (чек-лист + «Добавить пункт»).
- Примечание (многострочное).
- Снизу — «Удалить дело» (в режиме редактирования).

### 5.3. Всплывающий выбор даты
- Мини-календарь месяца с выделением сегодня. Тап по дню — выбрать и закрыть.

### 5.4. Выбор дела для привязки
- Список будущих дел с датой (цветная метка + диапазон). Тап — выбрать.

### 5.5. Настройки
- Тумблер «Автоперенос невыполненного на сегодня» (по умолчанию выкл).
- Тема (системная/светлая/тёмная).
- Первый день недели (Пн/Вс), по умолчанию Пн.
- (Зарезервировано под будущее: экспорт/импорт данных.)

## 6. Навигация

Главный экран → «+» или тап по делу → Карточка дела (full-screen route). Внутри карточки — всплывающие оверлеи (дата, выбор дела). Настройки — отдельный экран из шапки. Переключатель вида календаря — выпадающее меню на месте, без перехода.

## 7. Скоуп

### v1 (делаем сейчас)
Всё из разделов 3–6: модель дел, секции вчера/сегодня/завтра, ручной перенос + тумблер автопереноса, календарь (неделя/две/месяц) с дорожками, многодневные дела, жёсткие каскадные зависимости с ручным снятием, подпункты, напоминания (опц.), цвета, примечания, тёмная тема, RU-локализация. Только Android, только локально.

### v2 (отложено, не реализовывать сейчас)
- **Desktop-версия (Flutter desktop) + синхронизация только для себя** (например, через файл/папку или простой self-hosted механизм) — приоритет невысокий.
- Повторяющиеся дела (ежедневно/по дням недели).
- Сложные зависимости (несколько родителей, режим «предложить сдвинуть» вместо молчаливого).
- Быстрое создание дела из естественной фразы (парсинг «купить корм во вторник в 18:00») — опционально, возможно с ИИ.
- Ручная сортировка дел внутри дня перетаскиванием.
- Заметки как отдельная сущность (пока сознательно НЕ делаем).

## 8. Нефункциональные требования

- Плавность на крупном экране, длинные списки — ленивые (`ListView.builder`).
- Все даты — локальная таймзона, хранение даты без времени отдельно от времени дня.
- Округлять и форматировать даты по-русски («20 июня», «день 2 из 4»).
- Данные не теряются между запусками (миграции Drift при изменении схемы).
- Без аналитики и сетевых запросов.
