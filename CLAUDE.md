# CLAUDE.md — DayLane

Инструкции для Claude Code по автономной реализации приложения **DayLane**. Полное описание поведения и модели данных — в `SPEC.md` (читать первым, это источник истины).

---

## Что это

Локальное Android-приложение на Flutter для дел с тремя горизонтами (вчера/сегодня/завтра) и календарём, где многодневные дела показаны полосами с дорожками и жёсткими зависимостями. Без сервера, без аккаунтов, без ИИ, без сети.

Имя продукта: `DayLane` (вынести в одну константу `kAppName` + использовать в `applicationName`/манифесте, чтобы переименование было тривиальным). Bundle id: `world.misht.daylane` (поправить при желании).

---

## Стек

- Flutter (stable), Dart.
- Drift (SQLite) — слой данных.
- Riverpod — состояние.
- flutter_local_notifications + timezone — напоминания.
- intl / ARB — локализация (RU сейчас, задел под EN).
- Material 3, светлая + тёмная темы.

Никаких лишних зависимостей. Никаких сетевых пакетов.

## Структура проекта (ориентир)

```
lib/
  main.dart
  app.dart                 // MaterialApp, темы, локализация
  core/
    constants.dart         // kAppName, палитра цветов, дефолты
    date_utils.dart        // работа с датами без времени, "день N из M", формат RU
  data/
    db.dart                // Drift database, таблицы Task/Subtask, миграции
    task_dao.dart
    subtask_dao.dart
  domain/
    models.dart            // доменные типы, enum'ы (TaskKind, ReminderRule)
    scheduling.dart        // присутствие в дне, секции вчера/сегодня/завтра
    lanes.dart             // жадная укладка полос в дорожки
    dependencies.dart      // каскадный сдвиг, проверка циклов
    carry_over.dart        // перенос и автоперенос
  features/
    home/                  // главный экран: секции + календарь
    task_editor/           // карточка дела + всплывашки даты и выбора дела
    calendar/              // виды неделя/две/месяц, отрисовка полос
    settings/
  services/
    notifications.dart     // планирование/отмена локальных уведомлений
```

Структуру можно корректировать, но логику (scheduling/lanes/dependencies/carry_over) держать в `domain/` отдельно от UI и покрыть юнит-тестами.

## Порядок работы (вехи)

Делать инкрементально, после каждой вехи — рабочая сборка.

1. **Каркас.** Проект, темы (свет/тёмная), RU-локализация, навигация, константы и палитра.
2. **Слой данных.** Drift: таблицы Task/Subtask, DAO, миграции, сидинг тестовыми делами для отладки.
3. **Доменная логика (с тестами).** `scheduling` (присутствие в дне, секции), `lanes` (укладка), `dependencies` (каскад + запрет циклов), `carry_over`. Покрыть юнит-тестами — это ядро, баги тут дороже всего.
4. **Главный экран.** Три сворачиваемые секции со счётчиками (по умолчанию раскрыта «Сегодня»), строки дел с чипами и подпунктами, перенос «→ сегодня» и «перенести всё».
5. **Карточка дела.** Создание/редактирование: сегмент день/период, всплывающий календарь дат, тумблер «Начать после дела» с длительностью и вычисляемым концом, выбор родительского дела (только будущие/многодневные, без циклов), цвет, подпункты, примечание, удаление.
6. **Календарь.** Виды неделя/две недели/месяц(скользящий), полосы по сетке, дорожки, сегменты на границах недель, переключатель вида.
7. **Напоминания.** flutter_local_notifications: план/отмена при сохранении/сдвиге/удалении. По умолчанию выключены.
8. **Настройки.** Тумблер автопереноса (по умолчанию выкл), тема, первый день недели.
9. **Полировка.** Пустые состояния, анимации сворачивания, тёмная тема на всех экранах, проверка на крупном экране.

## Ключевые правила реализации (из SPEC, не нарушать)

- Время дня (`timeOfDay`) допустимо **только** для однодневных дел; у периода — null.
- Напоминания по умолчанию **выключены**.
- При включённой привязке источник истины — `durationDays`; Начало и Конец вычисляются (`start = parent.end + 1`, `end = start + duration - 1`), оба залочены.
- Сдвиг по зависимости — **автоматический и молчаливый**, каскадом; снятие привязки — **только вручную**. Циклы запрещены.
- Многодневное дело присутствует в **каждом** своём дне; одна галочка закрывает всё дело; полосу в календаре при выполнении **не укорачивать**.
- Автоперенос — **только** если включён тумблер в настройках; по умолчанию выкл.
- Укладка дорожек — автоматическая (жадный алгоритм по startDate), без ручного перетаскивания.

## Чего НЕ делать в v1

- Никакого ИИ, сети, аккаунтов, синхронизации.
- Никаких отдельных «заметок» как сущности.
- Никаких повторяющихся дел и множественных родителей у зависимости.
- Не реализовывать desktop (это v2).

## Definition of Done (v1)

- Приложение собирается в release APK и ставится на Android.
- Главный экран, карточка дела, календарь и настройки работают согласно `SPEC.md`.
- Доменная логика (scheduling, lanes, dependencies, carry_over) покрыта юнит-тестами, тесты зелёные.
- Данные сохраняются между запусками; миграции не теряют данные.
- Тёмная и светлая темы корректны на всех экранах; интерфейс на русском.
- Нет сетевых запросов и лишних разрешений в манифесте (кроме нужного для уведомлений).
